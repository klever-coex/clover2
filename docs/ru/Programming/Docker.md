## Введение

При разработке любых приложений, будь то веб-сайт, бэкенд или даже какой нибудь алгоритм для робота, всегда есть зависимости (библиотеки, файлы настроек).
При написании какой либо программы вы можете поставить все необходимые библиотеки и пакеты себе на компьютер, но при таком подходе рано или поздно можно столкнуться с следующими проблемами:

* Конфликт версий. Например написанная вами программа использует Python 3.12, а программа написанная ранее работает только с Python 3.8. Конечно вы можете установить себе обе версии Python, но в таком случае придётся при каждом использовании выбирать конкретную версию, настраивать переменные окружения и следить, чтобы одна программа случайно не «сломала» другую. Чем больше проектов — тем больше путаницы.

* «Работает у меня, но не работает у тебя». Вы запустили проект на своём компьютере — всё отлично. Передали другу или выложили на сервер — и вдруг появляются ошибки: другая операционная система, другие версии библиотек, не хватает каких-то файлов.

* Сложная установка. Иногда, чтобы запустить проект, нужно выполнить десятки команд команд: установить базы данных, настроить сервер, прописать пути. Для новичка или человека из другой области это может показаться кошмаром и только на запуск уйдет много времени.

* Захламление системы. Со временем компьютер превращается в «свалку» из разных версий программ, пакетов и сервисов, которые разбросанны по всей системе.

Чтобы решить эти проблемы, придумали контейнеризацию. Самый популярный инструмент для этого — [Docker](https://www.docker.com/).

Идея Docker очень простая:
вместо того чтобы устанавливать всё прямо на компьютер, мы «упаковываем» программу вместе со всеми её зависимостями в специальный контейнер.

Контейнер — это как коробка, внутри которой:
* сама программа
* нужная версия Python/ROS/OpenCV
* библиотеки
* настройки
* всё необходимое для запуска

Этот контейнер изолирован от остальной операционной системы. По сути, Docker позволяет запускать программы так, будто вы открываете и закрываёте коробки: открыл — всё работает, закрыл — как будто ничего и не было.

Но чтобы каждый раз не настраивать контейнер вручную, в Docker используют образы (images).
Образ — это как шаблон или «чертёж» нашей коробки. В нём заранее описано, какие программы установить, какие файлы скопировать и какую команду запустить.
Мы один раз собираем такой образ, а потом можем создавать из него сколько угодно контейнеров.

Самое удобное — образ можно сохранить и передать другу или выложить в интернет. Тогда друг просто скачает его и запустит, и приложение будет работать точно так же, как у вас, без долгой установки и настройки.

На этом этапе может показаться, что Docker — это обычная виртуальная машина. Но на самом деле это не совсем так. В отличие от виртуальной машины, Docker не запускает вторую операционную систему внутри вашей. Он создаёт лёгкое окружение только для вашей программы, с нужными библиотеками и настройками. Благодаря этому контейнеры используют ресурсы компьютера очень эффективно — не тратя процессор на лишнюю операционную систему, а только на то, что нужно приложению.

Docker помогает разработчикам избегать конфликтов версий, долгой настройки и «мусора» на компьютере. С его помощью можно упаковать программу со всеми зависимостями в контейнер, запускать её одной командой и делиться готовым образом с другими. Проще говоря, Docker превращает сложную установку в удобное «открытие коробки», которое одинаково работает у всех, при этом не затрачивая дополнительные ресурсы.

## Установка
Для установки Docker на устройство вам потребуется доступ в интернет и права администратора в терминале.

### Ubuntu
Оригинал статьи установки [тут](https://docs.docker.com/engine/install/ubuntu/).

Для установки Docker на операционную систему Ubuntu вам потребуется открыть терминал и выполнить следующие команды.

#### Обновление системы и установка curl
``` bash
sudo apt update
sudo apt install curl
```

#### Запуск скрипта установки
``` bash
curl -fsSL https://get.docker.com | sudo sh
```
