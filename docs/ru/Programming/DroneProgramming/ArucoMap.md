# Создание карты маркеров

<br>

## Формат YAML

> Не знакомы с YAML? См. [краткую памятку по YAML](../../UsefulInfo/YAML).

Карта описывается в формате YAML — это современный формат описания данных, который позволит без труда масштабировать набор параметров.
```yaml
name: simulation          # Имя карты (по умолчанию название файла)
default_size: 0.36        # Размер маркера по умолчанию в метрах (будет применяться ко всем маркерам у которых не указан параметр size)

markers:
  - id: 0                 # Уникальный ID маркера из словаря ArUco (обязательно)
    # size: 0.15          # Размер конкретного маркера (если поле отсутствет - применяется размер default_size)
    pose:
      x: 0.0              # Позиция X (м)
      y: 0.0              # Позиция Y (м)
      # z: 0.0            # Z — опционально, по умолчанию 0
    rot:
      yaw: -1.57079632    # Рыскание (рад) — опционально
      # roll: 0.0         # Крен (рад) — опционально
      # pitch: 0.0        # Тангаж (рад) — опционально
```

### Поля в `markers`

| Поле | Обязательно | По умолчанию | Описание |
|---|---|---|---|
| `id` | да | — | ID маркера из словаря ArUco |
| `size` | нет | `default_size` | Длина стороны маркера (м). Если не указан, берётся из `default_size`. Если нет ни того, ни другого — ошибка |
| `pose.x` | нет | `0.0` | Позиция по X в метрах |
| `pose.y` | нет | `0.0` | Позиция по Y в метрах |
| `pose.z` | нет | `0.0` | Позиция по Z в метрах |
| `rot.roll` | нет | `0.0` | Крен (рад) |
| `rot.pitch` | нет | `0.0` | Тангаж (рад) |
| `rot.yaw` | нет | `0.0` | Рыскание (рад) |
| `quat.x/y/z/w` | нет | `(0,0,0,1)` | Альтернатива `rot` — кватернион |
| `frame_id` | нет | `"{frame_id}_aruco_{id}"` | Кастомный TF frame для маркера |

Поворот задаётся либо через `rot` (RPY), либо через `quat` (кватернион). Одновременно использовать нельзя.

### Пример: базовая карта с default_size

```yaml
name: my_room
frame_id: map
version: 0
default_size: 0.30

markers:
  - id: 12
    pose:
      x: 1.0
      y: 0.5
  - id: 13
    pose:
      x: 1.0
      y: 1.5
```

Оба маркера получат `size=0.30` из `default_size`.

### Пример: индивидуальный размер и поворот

```yaml
markers:
  - id: 0
    size: 0.25
    pose:
      x: 0.0
      y: 0.0
      z: 0.5
    rot:
      yaw: 1.57
      pitch: 0.1
```

### Пример: кватернион вместо RPY

```yaml
markers:
  - id: 42
    pose:
      x: 2.0
      y: 3.0
    quat:
      x: 0.0
      y: 0.0
      z: 0.707
      w: 0.707
```

## Формат TXT (альтернативный)

:::{attention}
Крайне рекомендуем использовать yaml тк он является более перспективным и будет обретать новые возможности.
:::

Помимо YAML, карта может быть в текстовом формате с расширением `.txt`:

```
id size x y [z yaw pitch roll]
```

Каждая строка — один маркер. Пробел - разделитель. Строки, начинающиеся с `#`, игнорируются.

```
# id size x y z yaw pitch roll
0 0.36 0.0 0.0 0.0 -1.57 0.0 0.0
1 0.36 0.0 -1.0 0.0 -1.57 0.0 0.0
2 0.36 0.0 -2.0 0.0 -1.57 0.0 0.0
```

Поля `z`, `yaw`, `pitch`, `roll` опциональны — по умолчанию `0.0`.

## Как карта загружается

1. `clover2_map` ищет файл по пути из параметра `map_path`
2. По расширению определяется формат: `.yaml`/`.yml` или `.txt`
3. Для каждого маркера создаётся `Geometry_msgs/PoseWithCovariance` и статический TF фрейм
4. TF фрейм называется `{frame_id}_aruco_{id}` (например `map_aruco_0`)
5. Если у маркера указан `frame_id` — он используется вместо авто-имени
6. Карта публикуется по сервису `~/get_map` и в топик `~/map_update` при обновлении

## Размещение файла

Файлы карт находятся в `clover2_loc/clover2_map/map/`. Но так же для удобства карты можно класть в папку `/opt/clvoer2/map`. При запуске дрон будет сначала искать название карты в `/opt/clvoer2/map` а потом в `clover2_loc/clover2_map/map/`.

## Создание карты для помещения

1. Распечатайте ArUco маркеры (словарь 4X4_1000, размер соответствует словарю).
2. Разместите маркеры на потолке/стенах вдоль траектории полёта.
3. Измерьте позицию каждого маркера рулеткой относительно выбранного нуля.
4. Запишите в YAML: `id`, `pose.x/y/z`, `rot.yaw`.
5. Укажите `default_size` равный реальному размеру распечатанного маркера (размер маркера - это сторона черного квадрата).
6. Скопируйте файл в `clover2_loc/clover2_map/map/` и перезапустите `map_server`.
